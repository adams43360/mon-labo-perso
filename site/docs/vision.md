# Vision

Ce labo est un espace pour expérimenter sérieusement : chaque sujet est traité avec la même rigueur qu'un vrai produit — cadrage, analyse d'architecture, tests — mais avec la liberté de creuser des sujets sans enjeu business immédiat.

## Objectifs

- Construire une trace publique et navigable de ce qui est appris, au fil de l'eau
- Comparer des approches techniques concrètement (pas juste en théorie) via des démos fonctionnelles
- Garder une hygiène de Product Management même en solo : roadmap, priorisation, statut visible

## Ce que ce n'est pas

- Ni un dépôt de code jetable sans documentation
- Ni un blog théorique sans implémentation

## Le cycle de chaque expérience

Idée → Analyse d'architecture (ADR) → Implémentation + tests → Documentation → Publication → Mise à jour du roadmap.

Le code complet de chaque expérience vit dans [`experiments/`](https://github.com/adams43360/mon-labo-perso/tree/main/experiments) sur GitHub — ce site n'en est que la vitrine.
